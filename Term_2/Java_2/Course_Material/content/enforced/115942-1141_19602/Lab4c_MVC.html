Lab 4(c):  Model View Controller

Due: Friday, March 22, 2013 @ 11am

About this lab (30 points)

In the final part of the input/output assignment, you will use the principles of model-view-controller to take input from a GUI, populate a model, and write it through an object output stream to a file.

Part 1: Build the model (5 points)

Use the UserRegistration class which you built from Lab 4(b).  UserRegistration will be written to an external file, so it must have the following properties:

  1. The class must implement the Serializable interface.
  2. The class must contain basic user registration fields:  a username (String), a user ID (int), the user's first name and last name (Strings), a password (String), and an email address (String)
  3. Build a constructor for the class which will, at minimum, set the username, userID, and password.
  4. Build getters and setters for all fields except the password.  Password has a setter, no getter.
  5. The password must be transient.

You may reuse the class that you built in Lab 4(b), but...

  • Add another constructor which accepts all fields except password
  • Create the class in a package named my_name.assets

Part 2: Build the view (10 points)

Write a GUI class, UserRegistrationFrame, which extends JFrame.  This GUI class will be the visual representation of the UserRegistration class.  :

  1. Create the class in a package named my_name.gui
  2. It must use text fields for first name, last name, e-mail address, user name, and user ID.
  3. It must use a password field for the password.
  4. It must have two buttons:  OK and Cancel.  Listeners are implemented in the controller.
  5. Build a test class with a main() method which instantiates the frame and sets it to visible.  This is your only main() method.

Part 3: Build the controller (10 points)

Build the controller class.  The controller class does the following:

  1. UserRegistrationController should be in the my_name.control package.
  2. The controller implements the listener interface for the buttons.
  3. Implement listeners (ie, actionPerformed...) so that:
    1. When the user clicks Cancel on the view, the program terminates.
    2. When the user clicks OK on the view, the program will save the user registration details.
  4. When the user clicks OK...
    1. The controller creates a new instance of UserRegistration
    2. The values from the view are used to populate the instance.
    3. The new UserRegistration instance, filled with data from the view, is written to an output file using ObjectOutputStream.

Part 4: Bonus! (5 points)

Build two extra buttons to the View, Clear  and Read.  Clear will set all the fields to null (blank).  Read will read the file and populate the view with the contents of the file.

  • You may build the Clear functionality in the view.
  • You must build the Read functionality in the controller:
    • Create an instance of UserRegistration
    • Use an ObjectInputStream to read the object in and set it to the UserRegistration instance.
    • Use the values of the UserRegistration instance to set the fields in the view.
