About the assignment

Location services is a popular area at the moment, especially with the increasing power of smart phones.  Reverse geo-coding is part of this interesting field.

http://en.wikipedia.org/wiki/Reverse_geocoding

Basically, reverse geo-coding means taking a point location (latitude and longitude) and converting it back to a real address or place.  In the future, businesses will use this information to target deals in your area.  Imagine getting a text message with a coupon from The Gap because you’re in the American Eagle next door. There are many applications for its use.

There are a number of free location web services on the Internet that can be used to return useful information about a location.  Have a look at some of these:

http://www.geonames.org/export/reverse-geocoding.html

http://geocoder.ca/

In this assignment, you will create some of the parts of a reverse geo-coding application that can find interesting information about a point. You may be able to extend this into a full web application in the future (probably not in this class, though). For now, you will develop some of the parts.

Using your IDE, you will be required to develop the two classes listed below.

The Location class (Location.java)

Design and develop a class called Location.  The class should have:

  • Data fields
    • Longitude: a double representing the point’s longitudinal value.
    • Latitude: a double representing the point’s latitudinal value.
  • Methods
    • Accessor and mutator methods for each field
    • A toString() method that displays the location’s co-ordinates. For example,

Location: Latitude 44.3, Longitude -78.5

    • A constructor with two parameters, accepting double values for latitude and longitude and checking them for legal values.  If either of the values are not legal, the constructor should throw an IllegalArgumentException.

throw new IllegalArgumentException("Latitude is not in range [-90.0…90.0]" );

Question: How do I know which values are legal for latitude and longitude?

Answer: Legal values for latitude and longitude can be found at:

http://en.wikipedia.org/wiki/Latitude

http://en.wikipedia.org/wiki/Longitude

The LocationService class (LocationService.java)

Design and develop a class called LocationService. The class should have:

  • Data fields
    • None.
  • Methods
    • findCountry() - A method that takes a Location object and returns the country that the point is in.  The method signature should be

public String findCountry( Location location )

Note:

You DO NOT have to implement a real lookup, you can fake it (also called a 'mock-up'). For example, create a mock-up by looking the point up in an array of values and pretend to return a county.  Use whatever technique you want (e.g., a number of if-statements, a switch/case, etc.).  A good idea would be to look up the coordinates of a rectangle around a country to find the boundaries (You can use GoogleMaps here to help out).

Here is an example:

// Canada

// canada left -130.693359 bottom 42.875964 right -52.646484 top 74.116047

if(( location.getLatitude() >= 42.0 && location.getLatitude() <= 74.0 ) && ( location.getLongitude() >= -130.0 && location.getLongitude() <= -53 )){

return "Canada";

}

An example of using the method might be :

LocationService locationService = new LocationService();

String country = locationService.findCountry( new Location( 44.0, -78.0 ); //returns "Canada"
