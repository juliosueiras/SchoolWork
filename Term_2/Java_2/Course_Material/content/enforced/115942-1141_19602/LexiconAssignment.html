About this lab (30 points)

The field of artificial intelligence is growing and changing.  We are building more machines which are able to learn and interact with users in unique ways. A learning machine forms the basis of artificial intelligence.  To begin a study in artificial intelligence, you must establish a means of communicating and interacting with the artifice.  This goes deeper than computer programming - programming is the set of instructions under which the artifice operates.  You also need to be able to give the machine simple, non-programmatic instructions, have it understand those instructions, make decisions, and process them.  This is a tedious, and difficult process, but one starting point is to provide the machine with a lexicon. 

A lexicon is a catalog of words which the machine can use to help understand its directives.  When its issued a command, the machine can search its lexicon for the appropriate match and use that match to make a decision.  In this final assignment, you will use collections to be able to parse sentences and start to build a simple lexical framework for an artificial intelligence.

Part 1: Build the Lexcion (5 points)

The Lexicon is a Java class composed of several collections.  These collections represent the parts of speech which the machine can process.  There will be seven collections in your lexicon:

  • Determiners:  words like 'the', 'a', and 'an'
  • Nouns: persons, places, things ('rock', 'car', 'tank', 'you', 'me', etc.)
  • Verbs: action words ('roll', 'drive', 'fly', etc)
  • Adjectives: words which describe nouns ('smart', 'happy', 'hungry', etc.)
  • Adverbs: words which describe verbs (typically, words that end in '-ly')
  • Conjunctions: words which combine two sentences, like 'and', 'but', and 'or'
  • Miscellaneous: any other word that doesn't fit in these categories

These categories must be collections: you may choose ArrayList or LinkedList. 

Rules for the Lexicon class

  1. It must be Serializable
  2. It must have one field which describes the language.  The default language is English.
  3. It must be able to print any or all lexical categories to the System.out console.
  4. Initialize the categories with the following sentence:  "The brown fox jumped quickly over a lazy dog."  
    • 'the' and 'a' are Determiners
    • 'fox' and 'dog' are Nouns
    • 'jumped' is a Verb
    • 'brown' and 'lazy' are Adjectives
    • 'quickly' is an Adverb
    • 'over' is a preposition, to be placed in the 'Miscellaneous' collection

Part 3: Build the parser (10 points)

A parser is a class which breaks the sentence in to chunks and categorizes each word.  When the user types in a sentence, the parser will:

  • Break the sentence in to individual words
  • Analyze each word - search for the word in its lexicon. If it finds the word, it does nothing, and moves on to the next word.   If it does not find the word, it will:
    • ask the user what part of speech the word is
    • add the word to the appropriate category in the lexicon
    • sort the category
  • Note: the parser cannot distinguish singular from plural or different forms of the same verb (ie, 'sleeps', 'sleeping', 'slept').  They will all appear as entries in the lexicon.
  • The Parser is an interface with abstract methods - parseSentence(String s) and categorize(String word).  EnglishParser implements the parser and works with the English Lexicon.
