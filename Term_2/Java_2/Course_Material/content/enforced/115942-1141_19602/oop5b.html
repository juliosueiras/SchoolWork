Primitive Data Objects

In previous exercises, you've used the Integer and Double class when you've wanted to convert a String into an int or a double by using parse methods: the parseInt() method is part of the Integer class and the parseDouble() method is part of the Double class. These classes are called wrapper classes. Each primitive data type has its own wrapper class:

Primitive Type Wrapper Class
boolean Boolean
byte Byte
short Short
int Integer
long Long

All wrapper classes have the exact same name as the primitive type name, except that the wrapper class name starts with an upper-case letter.

If you examine the docs for wrapper classes like Integer and Double, you'll see that there are some constants for the minimum and maximum allowed value for the data type, as well as other methods that allow you to convert data to and from the primitive type.

The Character class includes many methods used for processing strings and characters, and we'll look at this class more closely at a later date.

Boxing and Unboxing; Autoboxing

Chapter 14.11 shows you how to turn primitives into objects and vice versa. Boxing is the term used for turning a primitive value into an object. Unboxing is the term for turning an object into a primitive value.

An example of boxing would be statements such as:

int num1 = 5; // create an int Integer

objNum1 = new Integer(num1); // create an Integer object

An example of unboxing would be statements such as:

Integer num1 = new Integer(1); // create a new Integer object

int num2 = num1.intValue(); // get the int value from this Integer object

Ever since Java SE 5.0, the Java compiler will automatically box or unbox a value or object for you, depending on the circumstances. For example:

Integer num1 = 1; // autobox the int value 1 into an Integer object

int num2 = num1 * 2;  // autounbox the num1 Integer object and multiply it's int value by 2

If you had to write this code before Java 5.0, you'd have to put:

Integer num1 = new Integer(1);

int num2 = num.intValue() * 2;

Autoboxing and autounboxing were added as a convenience to programmers, and you'll find them very helpful later on when working with objects, particularly when we start working with Collections.
