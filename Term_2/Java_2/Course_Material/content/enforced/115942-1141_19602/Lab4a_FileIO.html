About this lab

The Lightweight Directory Access Protocol, or LDAP, is an application protocol for accessing and maintaining distributed directory information services over an Intenet Protocol (IP) network.  It is used to identify members of the directory, by their name, organization, and other distinguishing factors, and to arrange those factors in a tree-like structure.

LDAP is one of the most commonly used protocols in today's business for maintaining user security information.  Many network security products use LDAP structures in their architecture. 

A typical LDAP structure starts at the root, the organization (characterized with the symbol 'o').  The organization is broken in to different organizational units ('ou').  The organizational units may be composed of users with IDs ('uid') and by named groups, or common names ('cn').  For example,

cn=swanma ou=trafalgar o=sheridan

cn=wangj uid=99221 ou=davis o=sheridan

cn=wangj uid=98765 ou=trafalgar o=sheridan

  2. The Java class must contain a main() method.
  3. Use the console (System.in) and a Scanner to take in the target file name (ie, the file to which you will write).
  4. If the file does not exist in the path, create a new file with the target name.  Otherwise use the target file name.
  5. Write a method which writes sample LDAP records in to the file as text (note: you may hardcode the records).
  6. Once the the records have been written, write a method which reads the sample LDAP records from the file, removes the LDAP tags (ie, cn=), and prints the results to the console window.
